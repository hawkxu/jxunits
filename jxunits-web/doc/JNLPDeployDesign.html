Access the JNLP deploy servlet through URL http://host.domain/context_path/servlet_path/URI, available URIs described below, replace the project with actual project name

The parameter followed (?) means optional

URI Method Parameter Response Description
/ GET HTML Home page
/_list GET JSON: project list Get deployed projects
/deploy/project/ GET HTML Deployment page for the project
/deploy/project/_start POST header: forceDeploy (?) deploy task number(?) Start project deploy
/deploy/project/_finish header: deployTask null Finish project deploy
/deploy/project/_cancel header: deployTask null Cancel project deploy
/deploy/project/_project GET header: deployTask (?) JSON: project Get project
POST header: deployTask
content: JSON
  {project, jnlpContent} 		JSON: project Update project
/deploy/project/_icons GET header: deployTask (?) JSON: icons Get project icons
POST header: deployTask
content: JSON
  {kind, iconContent} 		null Update project icon
/deploy/project/_libs GET header: deployTask JSON Get libraries of the project
/deploy/project/_lib POST header: deployTask
content: JSON
  {library, jarContent} 		JSON: library Update library
/project/ GET project jnlp file The two URIs has same effect
Note: the jnlp file name does not include version number
/project/project.jnlp
/project/_modules GET JSON: modules Get modules in the project
/project/path_to/jar_file GET jar file Get jar file of the project
